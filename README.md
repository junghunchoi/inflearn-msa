예) 123|456|789 이런식으로 저장하고, split을 통해 파싱한다.

문자열로 구분했기에 경로 컬럼이 정렬되어 있고 부모 댓글로 순서대로 자식 댓글도 정렬이 되어있어 해당 부분만 조회하기 때문에
커버링 인덱스가 사용되어 많은양의 데이터에도 빠르게 조회가 가능하다.
